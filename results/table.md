### Замечания по smoke-тесту

1. **Все 6 моделей обучаются и инферсятся корректно** после v32 рефакторинга —
   это была главная цель прогона. Метрики низкие из-за маленькой выборки
   (300-1000 молекул) и малого числа эпох (3-5).

2. **EGNN Vector всё ещё хуже скалярного EGNN на mu** (8.37 vs 1.52).
   Фиксы v32.6 (clamp) и v32.28 (нормализация pred) предотвращают NaN, но
   для сходимости нужно минимум 50-100 эпох на полном датасете. На 5 эпохах
   модель просто не успела выучить векторное представление диполя.

3. **EGNN+TDA показал себя лучше EGNN на alpha** (6.80 vs 10.20) —
   BatchNorm1d(tda_dim) (v32.7) действительно помогает. Это обнадёживающий
   сигнал для гипотезы, что `--tda_mode film` может превзойти concat на
   полном обучении.

4. **FCNN самый быстрый** (0.2 сек/эпоха) — не использует 3D-координаты,
   только агрегированные one-hot дескрипторы. **EGNN+TDA самый медленный**
   (52.5 сек/эпоха) — TDA-расчёт через GUDHI на CPU.

5. **FCNN robustness eval** (см. `results/smoke/robustness_fcnn.csv`) показал
   идентичные метрики при σ ∈ {0.0, 0.05, 0.10} — ожидаемо, т.к. FCNN
   не использует координаты, шум их не меняет. EGNN robustness eval не
   завершён из-за CPU-таймаута на 100 mol × 3 sigma.

## Замечания по воспроизводимости

Все результаты получены с фиксированным seed=42:
- Stratified split по HOMO-LUMO gap
- Train/Val/Test: 162063/20257/20259 молекул (80/10/10) на полном датасете
- Smoke-тест: 300-1000/100-200/100-200 (subsample для CPU-прогона)
- Target normalization: (y - mean) / std по train выборке
- Coordinate normalization: pos / cutoff (v32.5, раньше было pos / 5.0)

## Что нужно для финальных метрик

Smoke-тест доказал, что код v32 работает. Для получения метрик уровня
production нужно запустить на GPU (Kaggle T4 или локальная CUDA):

```bash
python src/run_all.py --models all --epochs 100 --batch_size 256 \
    --multi_gpu --num_workers 4 --device cuda
```

Ожидаемое время: 4-8 часов на все 6 моделей на 1 GPU T4.
