Геометрические симметрии (priors)

  • Трансляция — сдвиг не меняет свойства (инвариантность)
  • Вращение SO(3) — поворот сохраняет химию (эквивариантность)
  • Перестановка атомов — порядок не важен (инвариантность)

EGNN встраивает эти симметрии в архитектуру → быстрее сходимость, лучше точность.

Отличия моделей

  • FCNN → SchNet: добавляем translation + permutation инвариантность

  • EGNN Vector → EGNN Tensor: скалярный α → тензорный α через заряды (Часть B)
  • +TDA: добавляем 52D топологические признаки (Betti curves, persistence entropy)

Почему?

  • μ плохо сходится: заряды начинаются с zero-init → μ≈0 → нужно 188 эпох
  • α и gap лучше: векторное представление μ даёт более богатые признаки → mol_emb лучше описывает молекулу
  • test_loss в 2 раза лучше: 0.167 vs 0.344 — модель лучше обобщает

EGNN Tensor — Часть B (программа максимума)

Физически корректный расчёт через частичные заряды

μ = Σ qᵢ · (rᵢ − COM)                      # вектор R³
α = Σ qᵢ · (rᵢ − COM) ⊗ (rᵢ − COM)       # тензор R^(3×3)
α_iso = tr(α) / 3                          # скаляр (совместимость с Alchemy)

E(3)-эквивариантность 1-го и 2-го ранга

  • При вращении R: μ → R·μ, α → R·α·Rᵀ
  • При трансляции: μ и α не меняются
  • При перестановке: μ и α не меняются

Финальные выводы

  1. EGNN — лучшая модель для μ (0.284 Дебай). E(3)-эквивариантность работает.
  2. EGNN Vector — лучшая для α (0.354) и gap (0.0044). test_loss в 2 раза лучше EGNN.
  3. TDA не нужен — 54% фичей всегда ноль, ухудшение на +5-6% по всем метрикам. Валидный негативный результат.
  4. EGNN Tensor (Часть B) — физически корректный μ и α через заряды. Работает, но грубее MLP.
  5. Batch size — bs=256 лучше bs=1024 на 60% по μ MAE.
  6. SchNet остаётся лучшим baseline (μ MAE=0.131), но EGNN лучше для gap.
