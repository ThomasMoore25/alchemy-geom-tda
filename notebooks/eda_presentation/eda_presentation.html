Описание проекта

Задача

По 3D-структуре молекулы (координаты атомов) предсказать 3 квантово-механических свойства:

μ
Дипольный момент (Дебай) — полярность молекулы
α
Поляризуемость (Бор³) — реакция на электрическое поле
gap
HOMO-LUMO gap (Хартри) — электронная стабильность

Датасет Alchemy

202,579 органических молекул (H, C, N, O, F, S, Cl), 9-29 атомов. Свойства вычислены через DFT (B3LYP/6-31G(2df,p)).

Зачем геометрическое ML?

Свойства молекул не меняются при E(3)-преобразованиях (сдвиг, поворот, перестановка атомов). EGNN встраивает эти симметрии в архитектуру → быстрее сходимость, лучше точность, меньше данных нужно.

Как считается ошибка

Loss функция: MAE (Mean Absolute Error)

loss = MAE(pred_μ, target_μ) + MAE(pred_α, target_α) + MAE(pred_gap, target_gap) MAE = mean(|pred - target|)

Для векторного μ (EGNN Vector, EGNN Tensor)

# μ ∈ R³ (вектор), target = |μ| (скаляр из Alchemy) pred_μ_norm = pred_μ.norm(dim=-1).clamp(min=1e-4) # норма с защитой от NaN loss_mu = MAE(pred_μ_norm, target_μ)

Для EGNN Tensor (тензор α)

# α ∈ R^(3×3), target = tr(α)/3 (скаляр из Alchemy) pred_α_iso = trace(pred_α) / 3 loss_α = MAE(pred_α_iso, target_α) # + regularization на симметрию: loss += 0.01 * ||α - αᵀ||²

Метрики

MAE в физических единицах (Дебай, Бор³, Хартри) после денормализации. Target normalization: (y - mean) / std по train выборке.

Ключевые отличия

  • FCNN → SchNet: добавляем инвариантность к трансляциям и перестановкам
  • SchNet → EGNN: добавляем эквивариантность к вращениям (полная E(3))
  • EGNN → EGNN Vector: скалярный |μ| → векторный μ ∈ R³ через частичные заряды
  • EGNN Vector → EGNN Tensor: скалярный α → тензорный α ∈ R^(3×3) (Часть B)
  • +TDA: добавляем 52D топологические признаки

Параметры моделей: визуализация

Структура параметров EGNN

EGNN (952K параметров): ├── atom_embed: 7 × 128 = 896 ├── EGNN layers (×4): 4 × 227K = 908K │ ├── edge_mlp: Linear(129→32) + Linear(32→128) + Linear(128→128) │ ├── node_mlp: Linear(128→128) + Linear(128→128) │ └── coors_mlp: Linear(128→1) ├── final_norm: LayerNorm(128) = 256 ├── global_norm: LayerNorm(9) = 18 └── heads: ├── mu_head: Linear(137→128) + Linear(128→1) = 17,665 ├── alpha_head: Linear(137→128) + Linear(128→1) = 17,665 └── gap_head: Linear(137→128) + Linear(128→1) = 17,665

Почему EGNN Vector (968K) > EGNN Tensor (942K)?

EGNN Vector heads (50K): ├── charge_head: 16K (MLP) ├── alpha_skip + alpha_head: 18K (MLP) └── gap_head: 18K (MLP) → α предсказывается через MLP
EGNN Tensor heads (25K): ├── dipole_pol_head: 8K (MLP) └── gap_head: 18K (MLP) → α вычисляется физически: α = Σ qᵢ·(rᵢ−COM)⊗(rᵢ−COM) → не нужен alpha_head MLP

3D-визуализация: как каждая модель "видит" молекулу

FCNN (скалярный)

Атомы → one-hot + масса ↓ Mean/Max/Sum pooling ↓ MLP → [μ, α, gap]

Не использует 3D-координаты. Только типы атомов.

SchNet (translation + permutation)

Атомы → Embedding ↓ Расстояния ||rᵢ - rⱼ|| ↓ Continuous filter conv ↓ Pooling → MLP → [μ, α, gap]

Использует расстояния, но не направления.

EGNN (E(3) эквивариант)

Атомы → Embedding + координаты ↓ EGNN layers (update features) ↓ E(3)-эквивариантные признаки ↓ Pooling → MLP → [μ, α, gap]

Кодирует 3D-структуру эквивариантно.

EGNN Vector (вектор μ)

EGNN → признаки hᵢ ↓ qᵢ = MLP(hᵢ) — заряды ↓ μ = Σ qᵢ·(rᵢ−COM) ∈ R³ α = MLP(mol_emb) gap = MLP(mol_emb)

μ — вектор! α и gap через MLP.

EGNN Tensor ⭐ (вектор μ + тензор α) — Часть B

EGNN → признаки hᵢ ↓ qᵢ = MLP(hᵢ) — заряды (zero init) ↓ μ = Σ qᵢ·(rᵢ−COM) ∈ R³ (вектор, l=1) α = Σ qᵢ·(rᵢ−COM)⊗(rᵢ−COM) ∈ R^(3×3) (тензор, l=2) α_iso = tr(α)/3 (скаляр для Alchemy) gap = MLP(mol_emb)

Физически корректные μ и α через частичные заряды. E(3)-эквивариантность 1-го и 2-го ранга.

Почему?

  • μ плохо сходится: заряды начинаются с zero-init → μ≈0 → нужно 188 эпох для схождения
  • α и gap лучше: векторное μ через заряды даёт более богатый mol_emb — информация о полярности связей
  • test_loss в 2 раза лучше: 0.167 vs 0.344 — модель лучше обобщает

EGNN Tensor — Часть B (программа максимума)

Физически корректный расчёт через частичные заряды

μ = Σ qᵢ · (rᵢ − COM) ∈ R³

α = Σ qᵢ · (rᵢ − COM) ⊗ (rᵢ − COM) ∈ R^(3×3)

E(3)-эквивариантность 1-го и 2-го ранга

  • При вращении R: μ → R·μ (вектор), α → R·α·Rᵀ (тензор)
  • При трансляции: μ и α не меняются
  • При перестановке: μ и α не меняются

Финальные выводы

  1. EGNN — лучшая модель для μ (0.284 Дебай). E(3)-эквивариантность работает.
  2. EGNN Vector — лучшая для α (0.354) и gap (0.0044). test_loss в 2 раза лучше EGNN (0.167 vs 0.344).
  3. TDA не нужен — 54% фичей всегда ноль, ухудшение на +5-6%. Валидный негативный результат.
  4. EGNN Tensor (Часть B) — физически корректные μ и α через заряды. Работает, но грубее MLP.
  5. Batch size — bs=256 лучше bs=1024 на 60% по μ MAE.
  6. SchNet остаётся лучшим baseline (μ MAE=0.131), но EGNN лучше для gap.
